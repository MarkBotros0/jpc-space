# JPC Portal

Community management web app for **Jesus Project Community (JPC)** — a Christian student community organized into seasonal discipleship courses (e.g., "GBV 2026", "TW 2025").

## Stack

- **Next.js 16** (App Router, TypeScript strict, Server Actions)
- **Tailwind v4** + **shadcn/ui** (neutral, base-nova)
- **Prisma 7** + **PostgreSQL** (Neon-compatible) via `@prisma/adapter-pg`
- **NextAuth v5 (Auth.js)** with Credentials provider, JWT sessions
- React Hook Form + Zod, date-fns, lucide-react, nanoid

Package manager: **npm**.

## Roles

| Role | Scope | Capabilities |
|---|---|---|
| **Super** | Global | Creates seasons, assigns admins, manages users, global read |
| **Admin** | Season-scoped | Builds session calendar, creates assignments, manages groups within their seasons |
| **Leader** | Group-scoped | Reviews their group's submissions, marks attendance, adds engagement notes |
| **Student** | Current season + read-only history | Sees own current season (full) + previous seasons (read-only, no submission details) |
| **Mentor** | Global read-only | Pastoral read across all students/seasons |

A user has a global base `role` (UserRole enum), plus optional **scoped assignments** via `SeasonAdmin` and `GroupLeader` join tables — so a former Student can later be a Leader without losing history.

## Setup

### Prerequisites

- Node.js 20+ (tested on 24.14)
- npm 10+
- PostgreSQL 15+ (local or hosted — Neon works out of the box)

### Install & configure

```bash
npm install
cp .env.example .env       # then fill in DATABASE_URL and AUTH_SECRET
```

Generate `AUTH_SECRET` with:

```bash
openssl rand -base64 32
```

### Database

```bash
npm run db:migrate         # applies prisma/migrations
npm run db:generate        # generates Prisma client to src/generated/prisma
npm run db:seed            # populates realistic dev data (idempotent)
npm run db:studio          # opens Prisma Studio
```

### Dev server

```bash
npm run dev
```

Open <http://localhost:3000>.

## Project layout

```
prisma/
  schema.prisma                 # full domain model
  migrations/                   # autogenerated SQL migrations
  seed.ts                       # realistic dev data
src/
  app/
    layout.tsx, page.tsx        # root scaffolding
    api/auth/[...nextauth]/     # NextAuth route handler
  components/ui/                # shadcn primitives (add as needed)
  generated/prisma/             # Prisma client output (gitignored)
  lib/
    db.ts                       # PrismaClient singleton (pg adapter)
    auth.ts                     # NextAuth v5 config (Credentials + JWT)
    invites.ts                  # createInvite / acceptInvite server actions
    public-id.ts                # nanoid wrapper for Submission.publicId
    slug.ts                     # season code slugifier + validator
    rbac.ts                     # role/scope helpers
    storage/                    # Storage interface + Local/S3 drivers
    utils.ts                    # shadcn cn() helper
uploads/                        # local dev file storage (gitignored)
```
